# 贡献指南

感谢您对《记忆当铺》项目的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 📝 内容创作

#### 剧本扩展
- 完善后续集数的详细剧本（12集短剧版）
- 创作100-120章节版本（每章1800-2000字）
- 严格遵循"黄金三秒"和"结尾钩子"设计原则
- 补充场景细节和对话
- 优化现有剧本

**章节创作要求**：
- 每章字数控制在1800-2000字
- 开头必须有"黄金三秒"（50-100字）
- 结尾必须有"结尾钩子"（50-100字）
- 快节奏推进，避免拖沓
- 详见 [章节创作指南](docs/chapter_structure_guide.md)

#### 世界观扩展
- 补充世界观设定细节
- 创作背景故事
- 设计新的记忆技术

#### 角色发展
- 深化角色背景
- 创作角色外传
- 设计次要角色

### 🎨 视觉设计

- 角色概念设计
- 场景概念图
- 道具设计图
- 分镜头脚本
- 海报设计

### 📚 文档完善

- 翻译成其他语言
- 补充技术文档
- 编写教程和说明
- 整理参考资料

### 💡 创意建议

- 提出改进建议
- 报告问题或不一致之处
- 分享灵感和想法

## 贡献流程

### 1. Fork 项目

点击页面右上角的 "Fork" 按钮，将项目复制到您的账户。

### 2. 创建分支

```bash
git checkout -b feature/your-feature-name
```

分支命名规范：
- `feature/` - 新功能
- `fix/` - 修复问题
- `docs/` - 文档更新
- `design/` - 设计相关

### 3. 进行修改

- 遵循现有的文档格式
- 保持内容与世界观一致
- 确保逻辑自洽

### 4. 提交更改

```bash
git add .
git commit -m "描述您的更改"
```

提交信息规范：
- 使用清晰、简洁的描述
- 说明更改的内容和原因
- 一次提交只做一件事

### 5. 推送到 GitHub

```bash
git push origin feature/your-feature-name
```

### 6. 创建 Pull Request

- 在 GitHub 上创建 Pull Request
- 详细描述您的更改
- 等待审核和反馈

## 内容规范

### 剧本格式

遵循现有剧本格式：

#### 短剧版格式（12集）
```markdown
#### 场景X：场景名称-内/外景-时间

[场景描述]

**角色名**：（动作/情绪）对白内容
```

#### 章节版格式（100-120章）
```markdown
# 第X章：章节标题

## 【黄金三秒】
[开场钩子，50-100字]

---

## 正文
[主体内容，1600-1800字]

---

## 【结尾钩子】
[结尾悬念，50-100字]

---

**本章字数**：XXXX字
**关键词**：[关键词1]、[关键词2]、[关键词3]
```

详见 [章节创作指南](docs/chapter_structure_guide.md) 和 [章节示例](script/chapters_01_05.md)。

### 文档格式

- 使用 Markdown 格式
- 保持标题层级清晰
- 添加必要的目录和链接
- 使用合适的表情符号增强可读性

### 命名规范

- 文件名使用小写字母和下划线
- 使用有意义的描述性名称
- 例如：`episode_02_script.md`

## 内容要求

### 必须遵守

✅ 符合已有世界观设定  
✅ 保持角色性格一致  
✅ 逻辑自洽，无矛盾  
✅ 原创内容，不侵权  
✅ 尊重版权和道德规范

❌ 抄袭他人作品

❌ 包含暴力、色情等不当内容

❌ 低质量、敷衍的内容

## 审核流程

1. **提交审核**：创建 Pull Request
2. **初步检查**：检查格式和基本要求
3. **内容审核**：评估内容质量和一致性
4. **反馈修改**：根据反馈进行调整
5. **合并接受**：审核通过后合并
