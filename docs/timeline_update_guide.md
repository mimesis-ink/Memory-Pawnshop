# 时间线更新指南

> **适用对象**：所有参与章节创作的作者  
> **更新频率**：每创作5-10章更新一次  
> **维护文档**：[docs/timeline.md](timeline.md)

---

## 📋 为什么需要更新时间线？

### 核心价值
- ✅ **保持剧情连贯性**：避免时间矛盾和逻辑错误
- ✅ **方便多人协作**：所有作者都能看到故事进展
- ✅ **提升创作效率**：快速查阅关键事件，避免重复查找
- ✅ **质量控制**：确保故事节奏和时间安排合理

### 时间矛盾示例
❌ **错误案例**：
- 第20章：12月18日晚上，林深在家休息
- 第21章：12月18日下午，林深在记忆黑市
- **问题**：时间倒流，不合逻辑

✅ **正确案例**：
- 第20章：12月18日下午，林深在记忆黑市
- 第21章：12月18日晚上，林深回家整理线索
- **正确**：时间连贯，逻辑清晰

## 🔄 更新流程（三步法）

### 第一步：创作前查阅
**目的**：了解故事进展到哪里，避免时间冲突

**操作**：
1. 打开 `docs/timeline.md`
2. 查看最新章节的时间点
3. 确定你要创作的章节应该发生在什么时间

**示例**：
```markdown
# 查阅timeline.md最新记录
#### 12月20日（周三）- 第31-40章
- **20:00** - 计划记忆恢复
  - 章节：第32章

# 确定新章节时间
你要创作第33章，应该发生在12月20日20:00之后
或者12月21日凌晨
```

### 第二步：创作时遵循
**目的**：在创作过程中保持时间一致性

**操作**：
1. 在章节开头明确标注时间
2. 确保章节内事件的时间跨度合理
3. 与前一章的结尾时间衔接

**示例**：
```markdown
# 第33章：深夜行动

**时间**：12月20日（周三）23:00  
**地点**：地下诊所外

## 【黄金三秒】
夜幕下的巷子里，林深和苏晚盯着那扇生锈的铁门...
```

### 第三步：创作后更新
**目的**：将新创作的章节事件加入时间线

**操作**：
1. 打开 `docs/timeline.md`
2. 找到对应日期的章节
3. 添加新的事件条目
4. 更新文档开头的"最后更新"日期

**示例**：
```markdown
#### 12月20日（周三）- 第31-40章

... （已有内容）

- **23:00** - 抵达地下诊所
  - 地点：城郊废弃工业区
  - 人物：林深、苏晚
  - 内容：准备进行非法记忆恢复手术
  - 风险：可能被追踪，可能造成脑损伤
  - 章节：第33章
```

## 📝 事件条目标准格式

### 完整格式
```markdown
- **HH:MM** - 事件简述（3-8个字）
  - 地点：具体位置
  - 人物：主要参与人员
  - 内容：事件详细描述（1-2句话）
  - 影响：对剧情的推动作用（可选）
  - 章节：第X章
```

### 必填字段
- ✅ **时间**（HH:MM）：具体到小时和分钟
- ✅ **事件简述**：简洁明了
- ✅ **地点**：明确场景
- ✅ **章节**：标注章节编号

### 可选字段
- 人物（如果不言自明可省略）
- 影响（重大转折点才需要）

### 示例对比

**❌ 不规范**：
```markdown
- 林深做了一些事情
  - 第34章
```

**✅ 规范**：
```markdown
- **02:00** - 记忆恢复手术开始
  - 地点：地下诊所
  - 人物：林深、老记忆师
  - 内容：开始非法记忆恢复，过程极度痛苦
  - 章节：第34章
```

## 🎯 常见场景处理

### 场景1：章节跨越多个时间段
**情况**：一章内发生了多个重要事件，时间跨度较大

**处理方法**：分别添加多个事件条目

**示例**：
```markdown
#### 12月21日（周四）- 第35-45章

- **02:00** - 记忆恢复手术开始
  - 地点：地下诊所
  - 内容：老记忆师开始操作
  - 章节：第35章

- **03:30** - 首批记忆碎片恢复
  - 地点：地下诊所
  - 内容：林深看到陈煜开枪的画面
  - 章节：第35章

- **04:00** - 发现隐藏记忆
  - 地点：地下诊所
  - 内容：记忆中出现多年前救苏晚的画面
  - 章节：第35章
```

### 场景2：时间不确定或模糊
**情况**：某些事件发生在"深夜""凌晨"等模糊时间

**处理方法**：使用合理估算，标注"约"

**示例**：
```markdown
- **约03:00** - 杀手突袭地下诊所
  - 地点：地下诊所
  - 内容：陈煜的人追踪到位置，展开攻击
  - 章节：第38章
```

### 场景3：回忆或闪回
**情况**：章节包含回忆场景，时间在过去

**处理方法**：在当前时间线记录"闪回"，并在【序章：背景事件】补充

**示例**：
```markdown
# 当前时间线
- **04:00** - 闪回：多年前的救援
  - 内容：记忆中看到自己救苏晚的画面（实际发生在多年前）
  - 章节：第37章

# 序章部分
#### 多年前（具体年份待定）
- **林深救苏晚**（详细描述）
  - 章节：第37章揭示
```

### 场景4：多线叙事
**情况**：章节讲述多个人物的平行事件

**处理方法**：按时间顺序排列，标注人物

**示例**：
```markdown
- **14:00** - 林深查阅资料
  - 地点：苏晚的安全屋
  - 人物：林深
  - 章节：第40章

- **14:00** - 陈煜召开紧急会议
  - 地点：陈氏集团总部
  - 人物：陈煜、赵医生
  - 内容：讨论如何处理林深
  - 章节：第40章
```

## ⚠️ 常见错误及避免方法

### 错误1：时间跳跃过大
❌ **问题**：
- 第30章：12月19日晚上
- 第31章：12月22日早上
- **跳过**：整整两天，中间发生了什么？

✅ **解决**：
- 要么补充过渡章节
- 要么在第31章开头用简短文字说明这两天的情况

### 错误2：时间倒流
❌ **问题**：
- 第25章：12月20日下午
- 第26章：12月20日上午
- **错误**：时间倒流

✅ **解决**：
- 检查章节顺序，确保时间递进
- 如必须使用倒叙，需在章节开头明确标注"回溯"

### 错误3：事件时长不合理
❌ **问题**：
- 10:00 开始对话
- 10:05 完成复杂调查、跨城移动、深度交谈
- **错误**：5分钟完成不可能

✅ **解决**：
- 估算事件合理时长
- 复杂调查：至少2-4小时
- 跨城移动：根据距离，至少30分钟-2小时
- 深度对话：至少15-30分钟

### 错误4：忘记更新时间线
❌ **问题**：
- 创作了第40-45章
- 时间线还停留在第30章
- **后果**：后续作者不知道剧情进展

✅ **解决**：
- 每完成5-10章立即更新
- 创作时就在timeline.md旁边打开编辑器
- 养成"创作完成→立即更新"的习惯

## 🔧 实用技巧

### 技巧1：使用时间锚点
**方法**：在每个创作阶段设置固定时间锚点

**示例**：
- **第一阶段锚点**：12月17日18:00（林深醒来）
- **第二阶段锚点**：12月18日00:00（收到威胁）
- **第三阶段锚点**：12月20日09:00（陈煜找上门）

**好处**：其他事件都围绕锚点安排，不容易乱

### 技巧2：并行编辑
**方法**：创作时同时打开两个文档
- 左边：你正在创作的章节文件
- 右边：docs/timeline.md

**好处**：随时参考，随时更新，不会遗漏

### 技巧3：使用标记
**方法**：临时事件用 `[待确定]` 标记

**示例**：
```markdown
- **[待确定]** - 永生会首次提及
  - 地点：[待确定]
  - 章节：第45章
  - 备注：需要与前面章节协调具体时间
```

**后续**：确定后再补充完整信息

### 技巧4：定期检查
**方法**：每完成10章，通读一遍timeline.md

**检查内容**：
- [ ] 时间顺序是否合理？
- [ ] 有无时间矛盾？
- [ ] 事件密度是否适中？
- [ ] 关键转折是否都已记录？

- [ ] 章节开头标注时间
- [ ] 事件时长合理
- [ ] 与前后章节衔接

- [ ] 添加新事件到timeline.md
- [ ] 更新"最后更新"日期
- [ ] 检查无时间矛盾
- [ ] 提交更新（如使用版本控制）

## 🤝 多人协作注意事项

### 协作场景
如果多位作者同时创作不同章节：

1. **提前沟通**
   - 在团队群/文档中声明："我正在创作第X-Y章，时间段为12月Z日"
   - 避免多人创作同一时间段

2. **预留时间空间**
   - 不要把时间排得太满
   - 为其他作者预留插入空间

3. **版本控制**
   - 如使用Git，更新timeline.md前先pull最新版本
   - 避免覆盖他人更新

4. **冲突解决**
   - 如发现时间冲突，与相关作者沟通
   - 优先保护核心主线时间
   - 支线剧情可灵活调整

## ❓ 常见问题

### Q1: 我忘记更新时间线了，怎么办？
A: 尽快补上。回顾你创作的章节，提取关键事件和时间，添加到timeline.md。

### Q2: 我发现timeline.md里有错误，可以改吗？
A: 可以，但需谨慎：
- 如果是明显错误（打字错误、时间笔误），直接修改
- 如果涉及剧情调整，需与团队讨论
- 修改后在文档顶部记录修改日期和原因

### Q3: 时间线太详细会不会限制创作自由？
A: 不会。时间线记录的是"已发生"的事件，为你提供参考坐标。在这些坐标之间，你有完全的创作自由。

### Q4: 如果我想创作"时间模糊"的章节（如梦境、回忆）？
A: 在timeline.md中标注清楚：
```markdown
- **时间模糊** - 林深梦境
  - 内容：梦中看到模糊的记忆片段
  - 备注：非现实时间线
  - 章节：第XX章
```

### Q5: 更新频率建议是多久？
A: 
- **理想**：每完成1章立即更新（最精确）
- **标准**：每完成5章更新一次（平衡效率和准确性）
- **最低**：每完成10章必须更新（避免脱节）

## 🎯 最终提醒

- ✅ 它帮助你避免迷路
- ✅ 它让团队步调一致
- ✅ 它提升故事质量

**但请记住**：
- ❌ 不要被时间线束缚创意
- ❌ 不要为了填充时间线而写流水账
- ❌ 不要忽视人物情感只追求时间准确

**最佳实践**：
灵活运用时间线，服务于故事本身。时间线是工具，故事才是核心。
